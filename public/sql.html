SQL - Structured Query Language

SQL (Structured Query Language) is a standard language for storing, manipulating, and retrieving data in relational database management systems. It is used by database administrators, developers, and data analysts to interact with databases and perform various operations on data.

Key Features

  • Data Definition: Create, modify, and delete database objects
  • Data Manipulation: Insert, update, delete, and query data
  • Data Control: Manage access permissions and security
  • Transaction Control: Ensure data consistency and integrity
  • Portable: Works across different database systems

Basic SQL Commands

-- Creating a table CREATE TABLE employees ( id INT PRIMARY KEY, first_name VARCHAR(50), last_name VARCHAR(50), email VARCHAR(100) UNIQUE, hire_date DATE, salary DECIMAL(10,2) ); -- Inserting data INSERT INTO employees (id, first_name, last_name, email, hire_date, salary) VALUES (1, 'John', 'Doe', 'john.doe@company.com', '2023-01-15', 75000.00); -- Querying data SELECT first_name, last_name, salary FROM employees WHERE salary > 50000; -- Updating data UPDATE employees SET salary = 80000.00 WHERE id = 1; -- Deleting data DELETE FROM employees WHERE id = 1;

SELECT Statements

-- Basic SELECT SELECT * FROM employees; -- Selecting specific columns SELECT first_name, last_name, email FROM employees; -- Filtering with WHERE SELECT * FROM employees WHERE salary > 60000; -- Sorting with ORDER BY SELECT first_name, last_name, salary FROM employees ORDER BY salary DESC; -- Limiting results SELECT * FROM employees LIMIT 10; -- Using aliases SELECT first_name AS "First Name", last_name AS "Last Name", salary AS "Annual Salary" FROM employees;

Joins

-- INNER JOIN SELECT e.first_name, e.last_name, d.department_name FROM employees e INNER JOIN departments d ON e.department_id = d.id; -- LEFT JOIN SELECT e.first_name, e.last_name, d.department_name FROM employees e LEFT JOIN departments d ON e.department_id = d.id; -- RIGHT JOIN SELECT e.first_name, e.last_name, d.department_name FROM employees e RIGHT JOIN departments d ON e.department_id = d.id; -- FULL OUTER JOIN SELECT e.first_name, e.last_name, d.department_name FROM employees e FULL OUTER JOIN departments d ON e.department_id = d.id;

Aggregate Functions

-- Counting records SELECT COUNT(*) FROM employees; -- Average salary SELECT AVG(salary) FROM employees; -- Maximum and minimum salary SELECT MAX(salary), MIN(salary) FROM employees; -- Sum of salaries SELECT SUM(salary) FROM employees; -- Grouping data SELECT department_id, COUNT(*) as employee_count, AVG(salary) as avg_salary FROM employees GROUP BY department_id HAVING COUNT(*) > 5;

Subqueries

-- Subquery in WHERE clause SELECT first_name, last_name, salary FROM employees WHERE salary > (SELECT AVG(salary) FROM employees); -- Subquery in FROM clause SELECT dept_name, avg_salary FROM ( SELECT d.department_name as dept_name, AVG(e.salary) as avg_salary FROM employees e JOIN departments d ON e.department_id = d.id GROUP BY d.department_name ) AS dept_stats; -- Subquery in SELECT clause SELECT first_name, last_name, salary, (SELECT AVG(salary) FROM employees) as company_avg FROM employees;

Advanced SQL Features

-- Common Table Expression (CTE) WITH high_earners AS ( SELECT * FROM employees WHERE salary > 100000 ) SELECT * FROM high_earners; -- Window Functions SELECT first_name, last_name, salary, ROW_NUMBER() OVER (ORDER BY salary DESC) as salary_rank, LAG(salary) OVER (ORDER BY hire_date) as prev_salary FROM employees; -- CASE Statements SELECT first_name, last_name, salary, CASE WHEN salary < 50000 THEN 'Low' WHEN salary < 80000 THEN 'Medium' ELSE 'High' END as salary_category FROM employees;

Getting Started

To start with SQL:

  1. Install a database system (MySQL, PostgreSQL, SQLite)
  2. Learn basic SELECT, INSERT, UPDATE, DELETE commands
  3. Practice with JOINs and subqueries
  4. Understand database design principles
  5. Learn about indexes and performance optimization

Popular Database Systems

  • MySQL: Open-source relational database
  • PostgreSQL: Advanced open-source database
  • Oracle: Enterprise database system
  • SQL Server: Microsoft's database solution
  • SQLite: Lightweight, embedded database

Common Use Cases

  • Web Applications: User data, content management
  • Business Intelligence: Reporting and analytics
  • Data Warehousing: Large-scale data storage
  • E-commerce: Product catalogs, orders, customers
  • Financial Systems: Transactions, accounts, reporting
